# DB 미등록 제품 처리 기준

## 목적
바코드 조회 실패, 제품 DB 미등록, 원재료명/영양성분 부족 상황에서 챗봇이 추측하지 않고 적절히 추가 정보를 요청하도록 한다.

## DB 미등록 상황
다음 경우 DB 미등록 또는 정보 부족으로 판단한다.
- 바코드로 제품을 찾을 수 없음
- 품목제조보고번호가 없음
- 제품명만 있고 원재료명이 없음
- 영양성분 정보가 없음
- 알레르기 정보가 없음

## 응답 원칙
- “DB에 등록되지 않은 제품입니다”라고 명확히 말한다.
- 안전 여부를 단정하지 않는다.
- 원재료명, 알레르기 표시, 영양성분표 사진 업로드를 요청한다.
- 사용자가 알레르기나 질환이 있으면 정보 확인 전 섭취를 권하지 않는다.

## 기본 응답 예시
“현재 이 제품은 DB에서 확인되지 않습니다. 안전한 판단을 위해 원재료명, 알레르기 표시, 영양성분표 사진이 필요합니다.”

## 알레르기 사용자의 경우
“알레르기 정보가 확인되지 않아 안전하다고 판단할 수 없습니다. 원재료명과 알레르기 표시를 확인하기 전에는 섭취를 피하는 것이 좋습니다.”

## 당뇨/혈당 관리 사용자의 경우
“당류와 탄수화물 정보를 확인할 수 없어 혈당 관리 관점에서 판단하기 어렵습니다. 영양성분표 확인이 필요합니다.”

## 다이어트 사용자의 경우
“열량과 영양성분 정보가 없어 다이어트 적합성을 판단하기 어렵습니다. 영양성분표 사진을 업로드해 주세요.”

## 금지 표현
- “검색 결과가 없지만 괜찮아 보입니다”
- “일반적으로 안전합니다”
- “제품명상 문제 없어 보입니다”
- “비슷한 제품 기준으로 판단하면 됩니다”

## 추가 요청 항목
사용자에게 요청할 수 있는 정보:
- 제품명
- 바코드
- 품목제조보고번호
- 원재료명 사진
- 알레르기 표시 사진
- 영양성분표 사진
