# 의도별 응답 기준

## 목적
사용자의 질문 의도에 따라 어떤 데이터와 어떤 응답 방식을 사용할지 정의한다.

## can_i_eat
사용자가 제품을 먹어도 되는지 묻는 경우.

예시 질문:
- “이거 먹어도 돼?”
- “나 이거 먹어도 괜찮아?”
- “아이한테 먹여도 돼?”
- “땅콩 알레르기 있는데 먹어도 돼?”

필요 데이터:
- 사용자 프로필
- 알레르기
- 질환 또는 식단 조건
- 원재료명
- 알레르기 정보
- 영양성분

응답 방식:
- 결론을 먼저 말한다.
- 위험도를 함께 표시한다.
- 원재료명 또는 영양성분 근거를 제시한다.

## allergy_check
알레르기 성분 포함 여부를 묻는 경우.

예시 질문:
- “우유 들어있어?”
- “땅콩 성분 있어?”
- “알레르기 괜찮아?”

필요 데이터:
- 원재료명
- 알레르기 정보
- 사용자 알레르기 정보

응답 방식:
- 해당 성분 포함 여부를 명확히 말한다.
- 포함되어 있으면 섭취를 피하라고 말한다.
- 교차오염 가능성은 주의로 설명한다.

## nutrition_explain
영양성분 설명을 원하는 경우.

예시 질문:
- “영양성분 설명해줘”
- “단백질 많아?”
- “칼로리 어때?”
- “당류 높은 편이야?”

필요 데이터:
- 열량
- 단백질
- 지방
- 탄수화물
- 당류
- 나트륨

응답 방식:
- 수치를 설명한다.
- 사용자의 목표가 있으면 그 기준으로 해석한다.
- 단순 나열보다 의미를 설명한다.

## health_risk_check
질환이나 건강 상태 기준으로 판단하는 경우.

예시 질문:
- “당뇨인데 괜찮아?”
- “고혈압 있는데 먹어도 돼?”
- “다이어트 중인데 어때?”
- “고지혈증 있으면?”

필요 데이터:
- 사용자 건강 상태
- 관련 영양성분
- 원재료명
- 식품 유형

응답 방식:
- 건강 상태와 관련된 성분을 우선 설명한다.
- 위험하면 섭취 주의 또는 비추천으로 답한다.
- 의학적 진단은 하지 않는다.

## ingredient_explain
특정 성분 설명을 원하는 경우.

예시 질문:
- “이 성분 뭐야?”
- “말토덱스트린이 뭐야?”
- “아스파탐 들어있는데 괜찮아?”

필요 데이터:
- 원재료명
- 성분 설명 문서
- 사용자 건강 상태

응답 방식:
- 성분의 일반적 역할을 설명한다.
- 사용자 상태와 관련된 주의사항을 말한다.
- 과장된 위험 표현은 피한다.

## missing_product
제품 정보가 없는 경우.

예시 질문:
- 바코드 조회 실패
- DB에 제품 없음
- 제품 정보 없음

필요 데이터:
- 바코드
- 제품명
- 사진 업로드 여부

응답 방식:
- DB에 없다고 말한다.
- 판단을 보류한다.
- 원재료명/영양성분표 업로드를 요청한다.
